## Интерфейсы в Go — подробно и структурированно

---

### 1. Что такое интерфейс?

**Интерфейс** в Go — это **абстрактный тип**, который определяет **набор методов** (поведение), но не реализует их.

Ключевая идея: **"Если тип реализует все методы интерфейса, он автоматически удовлетворяет этому интерфейсу"** (duck typing, но проверяемое на этапе компиляции).

```go
type Writer interface {
    Write([]byte) (int, error)
}

// Любой тип с методом Write автоматически реализует Writer
type MyWriter struct{}
func (m MyWriter) Write(data []byte) (int, error) {
    return len(data), nil
}

// Можно использовать
var w Writer = MyWriter{}  // ✅ работает
```

**Зачем нужны:**
- **Абстракция** — код работает с поведением, а не с конкретными типами
- **Тестирование** — легко подменять реализации (моки)
- **Слабая связанность** — зависимости через интерфейсы, а не конкретные типы
- **Полиморфизм** — разные типы можно обрабатывать единообразно

---

### 2. Приведение типов (утверждение)

Для интерфейсов работает **утверждение типа (type assertion)**:

```go
var any interface{} = "hello"

// Один параметр — если тип не совпадает, будет паника
str := any.(string)

// Два параметра — безопасно
str, ok := any.(string)  // ok = true
num, ok := any.(int)     // ok = false, num = 0
```

**Синтаксис:** `x.(T)` где `x` — интерфейс, `T` — конкретный тип или другой интерфейс.

**Для нескольких вариантов — type switch:**

```go
switch v := x.(type) {
case string:
    fmt.Println("строка:", v)
case int:
    fmt.Println("число:", v)
default:
    fmt.Printf("неизвестный тип: %T\n", v)
}
```

### 3. Внутреннее строение (как устроен интерфейс под капотом)

В Go интерфейс — это структура из двух указателей (на 64-битной системе — 16 байт):

```go
type iface struct {
    tab  *itab          // таблица методов + информация о типе
    data unsafe.Pointer // указатель на сами данные
}
```

**Для пустого интерфейса `interface{}` — структура `eface`:**
```go
type eface struct {
    _type *_type        // информация о типе
    data  unsafe.Pointer // указатель на данные
}
```

**Что хранится в `itab`:**
- Тип интерфейса
- Конкретный тип данных
- Таблица методов (связка методов интерфейса с методами конкретного типа)

**Важно:** 
- Интерфейс хранит **пару (тип, значение)**
- Интерфейс равен `nil` только если **и тип, и значение равны nil**

```go
var x interface{} = nil     // x == nil
var y *int = nil
var z interface{} = y       // z != nil (тип есть, хотя значение nil)
```

### 4. Тип интерфейса

Интерфейс — это **тип**. Его можно использовать везде, где ожидается тип:

```go
// Как параметр функции
func Process(w io.Writer) { ... }

// Как поле структуры
type Handler struct {
    Logger interface{ Log(string) }
}

// Как возвращаемое значение
func NewReader() io.Reader { ... }
```

**Интерфейсы могут быть встроены друг в друга:**

```go
type Reader interface { Read(p []byte) (n int, err error) }
type Writer interface { Write(p []byte) (n int, err error) }

type ReadWriter interface {
    Reader
    Writer
}
```

### 5. Type Assertion (подробно)

**Три формы утверждения типа:**

#### 5.1 На конкретный тип
```go
var i interface{} = "hello"
s := i.(string)        // если i не string — panic!
s, ok := i.(string)    // безопасно, ok = false если не тот тип
```

#### 5.2 На другой интерфейс
```go
var r io.Reader = strings.NewReader("hello")
// Проверяем, реализует ли r также io.Writer
w, ok := r.(io.Writer)  // ok = false, т.к. *strings.Reader не реализует Write
```

#### 5.3 Type switch
```go
func inspect(v interface{}) {
    switch v := v.(type) {
    case string:
        fmt.Println("string:", v)
    case int:
        fmt.Println("int:", v)
    case fmt.Stringer:
        fmt.Println("Stringer:", v.String())
    default:
        fmt.Printf("unknown: %T\n", v)
    }
}
```

### 6. Что такое пустой интерфейс? И зачем он используется?

**Пустой интерфейс — `interface{}` (или `any` в Go 1.18+)** — это интерфейс без методов.

```go
var x interface{}  // может хранить значение ЛЮБОГО типа
```

**Внутреннее устройство:** `eface` (два слова: тип + данные).

**Где используется:**

#### 6.1 Функции, работающие с любыми типами
```go
func PrintAnything(v interface{}) {
    fmt.Printf("%v\n", v)
}
```

#### 6.2 Контейнеры с разнотипными данными
```go
var mixed []interface{} = []interface{}{42, "hello", 3.14, true}
```

#### 6.3 JSON с неизвестной структурой
```go
var data map[string]interface{}
json.Unmarshal([]byte(jsonStr), &data)
```

#### 6.4 Прокси-функции (логгеры, обёртки)
```go
func LogAndCall(fn func(interface{}), arg interface{}) {
    log.Println("calling with:", arg)
    fn(arg)
}
```

#### 6.5 Хранение значений в структурах без заранее известного типа
```go
type Event struct {
    Type string
    Data interface{}  // может быть любым: User, Order, etc.
}
```

### 7. Пустой интерфейс vs Пустая структура

**Очень важное различие!**

#### Пустой интерфейс: `interface{}` (16 байт)
```go
var x interface{} = 42
// Хранит и тип (int), и значение (42)
```

| Характеристика | Пустой интерфейс (`interface{}`) |
|----------------|----------------------------------|
| Размер | 2 слова (16 байт на 64-bit) |
| Содержит | **Тип + значение/указатель** |
| Может хранить | Любое значение любого типа |
| Используется | Когда тип неизвестен заранее |
| Пример | `json.Unmarshal`, `fmt.Println` |

#### Пустая структура: `struct{}` (0 байт)
```go
var s struct{}  // занимает 0 байт
```

| Характеристика | Пустая структура (`struct{}`) |
|----------------|-------------------------------|
| Размер | **0 байт** |
| Содержит | Ничего (это тип без полей) |
| Может хранить | Только значение `struct{}{}` |
| Используется | Когда нужно только наличие типа (сигналы, ключи мап) |
| Пример | `chan struct{}` для сигнализации, `map[string]struct{}` как множество |

**Главное отличие:**
- `interface{}` — **контейнер**, который может хранить значения любого типа
- `struct{}` — **пустой тип**, который занимает 0 памяти

**Пример использования `struct{}`:**
```go
// Канал для сигнализации (не передаём данные, только событие)
done := make(chan struct{})

// Множество (set) в Go
set := make(map[string]struct{})
set["apple"] = struct{}{}
_, exists := set["apple"]  // true
```

**Пример путаницы (чего делать не стоит):**
```go
var x struct{} = 42  // ❌ ОШИБКА! struct{} не может хранить 42
var y interface{} = 42  // ✅ ок
```

### Бонус: Приемники методов (value vs pointer)

При реализации интерфейса важно, на какой тип написаны методы:

```go
type Stringer interface {
    String() string
}

type MyType struct{}

// Метод на значении
func (m MyType) String() string { return "value" }

// Метод на указателе
func (m *MyType) StringPtr() string { return "pointer" }

// Использование:
var s Stringer = MyType{}   // ✅ ok (метод есть на значении)
var s Stringer = &MyType{}  // ✅ ok (метод есть и на указателе — Go вызывает автоматически)

// Если метод только на указателе:
type OtherType struct{}
func (o *OtherType) String() string { return "other" }

var s Stringer = OtherType{}  // ❌ ошибка! метод только на *OtherType
var s Stringer = &OtherType{} // ✅ ok
```

**Правило:** 
- Методы на значении доступны и для значения, и для указателя
- Методы на указателе — только для указателя

### Резюме

| Вопрос | Ответ |
|--------|-------|
| **Что такое интерфейс?** | Набор методов, определяющий поведение |
| **Приведение типов** | `x.(T)` — утверждение типа, `switch x.(type)` — переключатель |
| **Внутреннее строение** | Два слова: таблица методов + указатель на данные |
| **Тип интерфейса** | Может быть параметром, полем, возвращаемым значением |
| **Type assertion** | Извлечение конкретного типа из интерфейса |
| **Пустой интерфейс** | `interface{}` (any) — хранит значение любого типа |
| **Пустой интерфейс vs пустая структура** | `interface{}` — 16 байт, хранит тип+значение; `struct{}` — 0 байт, только сигнал |

- ‼️Реализация интерфейса должна быть не там где объявлен интерфейс, а там где он реализуется

- В Go интерфейсы реализуются неявно (duck typing).
- Если тип имеет все методы, которые требует интерфейс — он автоматически его реализует.

- Переиспользовать стандартные интерфейсы!
