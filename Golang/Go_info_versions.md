# Что такое Go

**Go** — это компилируемый, статически типизированный язык от Google, разработанный для создания высоконагруженных и масштабируемых систем. Он отличается простым синтаксисом, высокой производительностью, встроенной поддержкой конкурентности (горутины) и наличием сборщика мусора.

Go — **императивный** язык, но позволяет писать в **декларативном** стиле через функции высшего порядка.
- **Императивный** — вы говорите **КАК** сделать
- **Декларативный** — вы говорите **ЧТО** сделать

# История версий Go

## 1.18 (2022)
- **Дженерики (Generics)** — появление параметров типа (type parameters) для функций и типов.
- Новый синтаксис для объявления ограничений (constraints), например `[T any]`.
- Добавлен пакет `constraints` (позже перенесён в `golang.org/x/exp`).
- Исправление ошибки замыкания в цикле `for` (теперь переменные цикла создаются заново на каждой итерации).
- Новый флаг `-asan` (Address Sanitizer) для компилятора.

## 1.19 (2022)
- Уточнение документации (Godoc).
- Внедрение нового форматирования комментариев для документации (списки, заголовки, ссылки) — изменение в `go/doc`.
- Добавлены атомарные типы в `sync/atomic`, расширяющие возможности работы с памятью на уровне языка.
- Небольшие изменения в работе линтера.

## 1.20 (2023)
- **Преобразование срезов в массивы** — возможность преобразования среза в указатель на массив (например, `(*[4]byte)(slice)`), что позволяет компилятору проверять границы на этапе компиляции.
- Появилась возможность сравнивать структуры и массивы, содержащие поля, которые нельзя сравнивать (раньше это вызывало ошибку компиляции).
- Упаковка ошибок: теперь несколько ошибок могут быть объединены в одну.

## 1.21 (2023)
- **Встроенные функции для работы со срезами** — добавлены функции `min`, `max`, `clear` встроенные в язык (builtins).
- Пакет `cmp` (сравнение упорядоченных типов).
- Улучшение работы с дженериками: значительно улучшена производительность вывода типов (type inference), что сокращает необходимость явного указания типов.
- Поддержка инициализации карт (map) с необычными типами ключей.

## 1.22 (2024)
- **Исправление "цикла for" окончательное** — переменные цикла for теперь создаются на каждой итерации (окончательное исправление старой проблемы с замыканиями).
- **Целочисленный диапазон в range** — теперь можно писать `for i := range 10` (аналогично `for i:=0; i<10; i++`).
- Новый синтаксис маршрутов `net/http`: разрешены методы `GET /items/{id}` без необходимости регулярных выражений.

## 1.23 (2024)
- **Итераторы (Range over func)** — возможность итерации по функциям (range-over function iterators).
- Введение пакета `iter`, формализующего понятие последовательностей и итераторов.
- Теперь любой пользовательский контейнер может работать с `for range`, если реализует специальную функцию-итератор.
- Предварительная поддержка телеметрии (опционально).

## 1.24 (2025)
- **Встроенная поддержка OpenTelemetry** — на уровне рантайма добавлена поддержка трассировки через OpenTelemetry.
- Улучшена работа криптографических пакетов, использующих аппаратное ускорение (FIPS-режим).
- Появление новых встроенных функций для генерации случайных чисел (незначительные дополнения в `crypto/rand`).

## 1.25 (2025)
- **Улучшение конкурентного кода** — добавлен метод `WaitGroup.Go()` в пакет `sync`, который автоматически вызывает `Add(1)` перед запуском горутины и `Done()` по её завершению (синтаксический сахар для снижения ошибок).
- Мелкие исправления в работе `go test` для работы с race detector.

## 1.26 (2026)
- **Рекурсивные дженерики** — снято ограничение на рекурсивное инстанцирование дженериков. Теперь тип `T[T]` разрешён, что упрощает построение сложных структур (например, деревьев, комбинаторов).
- **Улучшенный new** — встроенная функция `new` теперь может принимать значение для инициализации: `new(int(100))`.
- **Новый GC "Green Tea"** стал стандартным (изменение в рантайме).
- **Экспериментальный пакет `simd/archsimd`** для доступа к SIMD-инструкциям.

# Основные директивы компилятора Go

- **`//go:build`**: Указывает условия, при которых файл должен быть включен в сборку (например, `//go:build linux && amd64`).
- **`//go:noinline`**: Запрещает компилятору встраивать (inline) функцию, что полезно для профилирования или отладки.
- **`//go:nosplit`**: Отключает проверку размера стека (stack-split check) для функции. Используется в низкоуровневом коде (например, в рантайме).
- **`//go:nowritebarrier`**: Запрещает барьеры записи, позволяя функции работать быстрее, но требует осторожности (используется для GC).
- **`//go:generate`**: Запускает внешние инструменты перед компиляцией (например, `go generate ./...`).

# Распространенные флаги go build (командная строка)

- **`-o <имя>`**: Задает имя выходного бинарного файла.
- **`-v`**: Выводит имена пакетов в процессе сборки.
- **`-x`**: Показывает команды, выполняемые компилятором.
- **`-race`**: Включает детектор гонок данных.
- **`-trimpath`**: Удаляет пути к исходным файлам из бинарника.
