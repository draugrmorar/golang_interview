ЗАМЫКАНИЕ (Closure)
    Замыкание — это функция, которая "запоминает" переменные из области видимости,
    где она была создана, даже после того, как эта область перестала существовать.

    func main() {
        message := "Привет"
        // Анонимная функция - замыкание
        sayHello := func() {
            fmt.Println(message) // "захватила" переменную message
        }
        sayHello() // Выведет: Привет
    }

    ГДЕ ИСПОЛЬЗУЮТ ЗАМЫКАНИЯ:
        - Фабрики функций
        - Инкапсуляция состояния (приватные переменные)
        - Callback функции
        - Middleware в веб-фреймворках

    КОГДА НЕ ИСПОЛЬЗОВАТЬ ЗАМЫКАНИЯ:
        В горячих циклах — могут быть накладные расходы
        Если переменные большие — они не освобождаются, пока живо замыкание
        Для простых операций — иногда проще передать параметры явно

    ЗОЛОТЫЕ ПРАВИЛА
        - Замыкание захватывает ССЫЛКУ, а не значение
          i := 0
          defer func() {
           fmt.Println(i)
          }();
          i = 5 // выведет 5
        - В цикле for — создавай копию или передавай параметр
          go func(val int) { ... }(i)
        - Замыкания полезны для фабрик и инкапсуляции
          multiplyBy(2)
        - Не злоупотребляй, если можно проще
          func add(a, b int) int // лучше, чем замыкание

defer
    defer имеет небольшой оверхед (порядка 50-100 наносекунд),
    поэтому если у вас код для высоконагруженной системы и 100 наносекунд имеют значение,
    то вам может быть выгодней не использовать defer

Switch
    Type switch — это конструкция для проверки типа интерфейсной переменной.
    Позволяет выполнять разный код в зависимости от конкретного типа значения.
            switch v := someInterface.(type) {
            case string:
                // v — string
                fmt.Println("Строка:", v)
            case int:
                // v — int
                fmt.Println("Число:", v)
            case MyStruct:
                // v — MyStruct
                fmt.Println("Моя структура:", v)
            case nil:
                fmt.Println("nil значение")
            default:
                fmt.Printf("Неизвестный тип: %T\n", v)
            }
    Ключевые моменты:
        Синтаксис: .(type) можно использовать только в switch
        Переменная: v получает значение с правильным типом в каждой ветке
        Порядок: ветки проверяются сверху вниз
        default: срабатывает, если ни один тип не подошел
        Обработка разных типов в функции, принимающей interface{}
        Работа с данными из JSON (где типы могут быть разными)
        Реализация полиморфизма без заранее известных типов
        Type switch работает только с интерфейсами
        Если переменная уже конкретного типа — компилятор ругнется
        Можно комбинировать несколько типов в одной ветке: case int, float64:
